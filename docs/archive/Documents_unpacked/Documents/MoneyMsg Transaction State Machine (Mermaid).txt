# Wallet Balance Locking Logic (Mermaid)

```mermaid
flowchart TD

A[Start Transfer] --> B{Sufficient Balance?}

B -->|No| F[Reject Transaction]

B -->|Yes| C[Move Funds to Locked Balance]

C --> D[Create Payment Link]

D --> E{Recipient Claims?}

E -->|Yes| G[Transfer to Recipient Wallet]

E -->|No - Expired| H[Return Funds to Sender]

G --> I[Transaction Completed]

H --> I
```

# Provider Event Flow (Mermaid)

Shows how webhook events from infrastructure providers advance the transaction state.

```mermaid
sequenceDiagram

participant User
participant MoneyMsg
participant Orchestrator
participant Plaid
participant TabaPay
participant Dwolla
participant Ledger

User->>MoneyMsg: Initiate payment
MoneyMsg->>Orchestrator: Create transaction

Orchestrator->>Plaid: Bank authentication
Plaid-->>Orchestrator: account_token

Orchestrator->>TabaPay: Card authorization
TabaPay-->>Orchestrator: authorization_result

Orchestrator->>Dwolla: Bank transfer request
Dwolla-->>Orchestrator: transfer_created

Dwolla-->>Orchestrator: transfer_completed
Orchestrator->>Ledger: Credit wallet
Ledger-->>Orchestrator: confirmation

Orchestrator-->>MoneyMsg: Transaction completed
```
